# RBG Planner

SLA-based autoscaler for [RoleBasedGroup](https://github.com/rolebasedgroup) Prefill/Decode disaggregated inference workloads.

The RBG Planner automatically scales Prefill and Decode roles within a RoleBasedGroup to meet TTFT (Time to First Token) and ITL (Inter-Token Latency) SLA targets using performance profiling data, load prediction, and real-time correction factors.

## Architecture

```
                    +-----------------+
                    |   Prometheus    |
                    | (inference      |
                    |  metrics)       |
                    +--------+--------+
                             |
                             | query metrics
                             v
+----------+        +--------+--------+        +------------------+
| Profiling |------->|   RBG Planner   |------->|  RBG Controller  |
| ConfigMap |        |                 |        |                  |
+----------+        | 1. Observe      |        | scales pods via  |
                    | 2. Predict      |  RBGSA | StatefulSets/    |
                    | 3. Compute      |  patch | Deployments      |
                    | 4. Scale        |        |                  |
                    +-----------------+        +------------------+
```

## How It Works

Each planning cycle:

1. **Observe** - Queries Prometheus for TTFT, ITL, request rate, ISL/OSL metrics from inference engines (SGLang/vLLM/Patio)
2. **Predict** - Uses configurable load predictor (ARIMA/Constant/Prophet) to anticipate upcoming demand
3. **Compute** - Interpolates profiling data to determine required prefill/decode replicas that satisfy SLA targets
4. **Scale** - Applies scaling decisions via RBGSA scale subresource (with fallback to direct RBG patch)

Correction factors continuously adjust predictions based on observed vs. expected performance.

## Prerequisites

- Kubernetes cluster with the RBG controller installed
- A RoleBasedGroup deployed with `prefill` and `decode` roles
- Prometheus monitoring stack (e.g., kube-prometheus-stack)
- Profiling data (generated by [inference-ext-cli](https://github.com/rolebasedgroup/inference-ext-cli))

## Installation

```bash
pip install rbg-planner
```

Or with Prophet support:

```bash
pip install "rbg-planner[prophet]"
```

## Quick Start

### Run as Container (Recommended)

Build and deploy as a sidecar role in your RBG:

```bash
docker build -t rbg-planner:latest .
```

The planner is typically deployed as a `planner` role within the RBG itself. Use `inference-ext-cli generate --enable-planner` to automatically inject it.

### Run Locally

```bash
export RBG_NAME=sglang-pd-inference
export RBG_NAMESPACE=default
export PROMETHEUS_ENDPOINT=http://localhost:9090
export PROFILE_RESULTS_DIR=./profiling-data
export TTFT_SLA=200
export ITL_SLA=20

rbg-planner
```

## Configuration

All configuration via environment variables (or CLI args as overrides):

| Variable | Default | Description |
|----------|---------|-------------|
| `RBG_NAME` | (required) | Name of the RoleBasedGroup |
| `RBG_NAMESPACE` | `default` | Kubernetes namespace |
| `PREFILL_ROLE_NAME` | `prefill` | Prefill role name in the RBG |
| `DECODE_ROLE_NAME` | `decode` | Decode role name in the RBG |
| `PROMETHEUS_ENDPOINT` | `http://prometheus-kube-prometheus-prometheus.monitoring.svc.cluster.local:9090` | Prometheus URL |
| `METRIC_SOURCE` | `sglang` | Metric source: `sglang`, `vllm`, or `patio` |
| `MODEL_NAME` | (empty) | Model name for Prometheus label filtering |
| `ADJUSTMENT_INTERVAL` | `180` | Seconds between scaling decisions |
| `MAX_GPU_BUDGET` | `8` | Maximum total GPUs (prefill + decode) |
| `MIN_REPLICAS` | `1` | Minimum replicas per role |
| `PREFILL_ENGINE_NUM_GPU` | `1` | GPUs per prefill engine instance |
| `DECODE_ENGINE_NUM_GPU` | `1` | GPUs per decode engine instance |
| `TTFT_SLA` | `500.0` | Target TTFT in milliseconds |
| `ITL_SLA` | `50.0` | Target ITL in milliseconds |
| `LOAD_PREDICTOR` | `arima` | Predictor: `constant`, `arima`, or `prophet` |
| `LOAD_PREDICTION_WINDOW_SIZE` | `50` | Data points in predictor window |
| `NO_CORRECTION` | `false` | Disable SLA correction factors |
| `PROFILE_RESULTS_DIR` | `/etc/rbg-planner/profiling` | Profiling data mount path |
| `PLANNER_PROMETHEUS_PORT` | `0` | Planner metrics port (0=disabled) |
| `NO_OPERATION` | `false` | Dry-run mode (observe only, no scaling) |

## Profiling Data

The planner requires profiling data that maps load characteristics to hardware requirements. This data is typically generated by `inference-ext-cli profile` and mounted as a ConfigMap.

**prefill_raw_data.json** - Maps input sequence length to TTFT and throughput/GPU:
```json
{
  "prefill_isl": [128, 256, 512, 1024, 2048],
  "prefill_ttft": [0.008, 0.015, 0.030, 0.060, 0.120],
  "prefill_thpt_per_gpu": [6000, 5000, 4000, 3000, 2000]
}
```

**decode_raw_data.json** - Maps KV cache usage and context length to ITL and throughput/GPU:
```json
{
  "x_kv_usage": [0.05, 0.1, 0.2, 0.3, 0.5, 0.7, 0.9],
  "y_context_length": [256, 512, 1024, 2048, 4096],
  "z_itl": [0.008, 0.010, 0.012, 0.015, 0.020, 0.025, 0.035],
  "z_thpt_per_gpu": [1200, 1100, 1000, 900, 800, 700, 500],
  "max_kv_tokens": 32768
}
```

## Prometheus Metrics

When `PLANNER_PROMETHEUS_PORT` is set (e.g., `9091`), the planner exposes:

| Metric | Description |
|--------|-------------|
| `rbg_planner_num_prefill_workers` | Current prefill replica count |
| `rbg_planner_num_decode_workers` | Current decode replica count |
| `rbg_planner_observed_ttft_ms` | Last observed TTFT |
| `rbg_planner_observed_itl_ms` | Last observed ITL |
| `rbg_planner_observed_request_rate` | Observed request rate (req/s) |
| `rbg_planner_predicted_request_rate` | Predicted request rate |
| `rbg_planner_predicted_num_prefill` | Predicted prefill replicas |
| `rbg_planner_predicted_num_decode` | Predicted decode replicas |
| `rbg_planner_p_correction_factor` | TTFT correction factor |
| `rbg_planner_d_correction_factor` | ITL correction factor |
| `rbg_planner_gpu_hours_total` | Cumulative GPU hours used |
